Resource Management

Managing Resources

A process refers to a program running on a computer, and the operating system (OS) oversees how these software processes utilize the system's resources. When software programs need to store data on a storage device like a hard disk, the OS ensures that multiple processes can save data simultaneously without conflicts. For instance, if three processes are running concurrently and regularly saving data, the OS handles their requests to prevent issues when the storage device is already occupied, enabling all three processes to successfully save their data.

Managing Processing Time

When multiple processes are running on a system, the operating system (OS) needs to allocate processing time among them. This is achieved through a scheduler, which executes a scheduling algorithm to determine how the processing time will be divided. The algorithm considers factors such as the importance of running processes and how interrupts are handled. When the OS switches between processes, the currently running process is paused, its state is saved, and it is placed in a queue. The scheduler then selects the next process to run based on its scheduling algorithm. In systems with multiple processors, the scheduler not only distributes processes among the cores but also ensures fair sharing of processing time between them.

Memory Management

The instructions and data needed for a process have to be available in main memory. To ensure this, the OS runs a memory manager program, which:

• Keeps track of what portions of memory have been allocated – memory is mapped as addresses, and the memory manager program tracks which addresses are allocated to each process
• Determines how much memory to allocate to each process
• Determines when a portion of memory should be available for a process

Managing Input/Output Devices

The operating system (OS) plays a crucial role in managing and utilizing input and output (I/O) devices. The OS handles I/O devices through the following methods:

• Recording device requirements: The OS keeps track of which device requires processor time. This allows the processor to communicate with the device without encountering conflicts.

• Prioritizing processes: The OS assigns priorities to processes based on the control signals sent and received by the I/O device. This ensures that processes requiring I/O operations are given appropriate attention.

• Handling critical requests: If an I/O device sends a more critical request while the system is executing a task, the OS has the capability to interrupt the current task and switch to the more important request. This allows the OS to prioritize critical I/O operations over less urgent ones.

Managing Secondary Storage

When files and folders reside on a secondary storage device connected to the operating system, the OS performs the following functions:

• Providing access to data and programs: The OS enables access to the data and programs stored on the secondary storage device and facilitates their loading into the main memory for execution.

• Tracking storage locations: The OS keeps a record of where programs and files are stored on the secondary storage device. This information allows the OS to efficiently retrieve and manage the stored data.

• Performing backup operations: The OS is responsible for executing backup operations to ensure data on the secondary storage device is safely preserved. This helps protect against data loss in case of hardware failures or other unforeseen events.

• Supporting a user-friendly interface: The OS provides a file management system that offers a user-friendly interface. This interface allows users to create, edit, and delete files and folders located on the secondary storage device. It simplifies file organization and facilitates user interactions with stored data.
